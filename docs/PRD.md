## 1. 文档信息

## 2. 产品概述

旅行意义记录 Agent 是一个面向个人旅行者的智能记录工具。它帮助用户在旅途中捕捉有意义的照片、物件、地点和片刻，并将其中的重要视觉记忆转化为可交互的 3D 记忆对象。用户可以为每个对象补充文字、语音、地点、时间、情绪、故事和标签，最终形成一个以地图、时间线和 3D 记忆库组织的私人旅行档案。

产品的核心不是泛旅行攻略，也不是普通相册，而是帮助用户把“我为什么记得这里”保存下来。照片负责记录画面，Agent 负责追问意义、组织上下文、生成 3D 纪念物，并在旅行结束后帮助用户回看、检索和分享。

## 3. 背景与机会

旅行中大量照片最终会沉没在相册里。用户拍下了风景、街角、路边小店、纪念品、建筑细节或偶遇的人和事，但事后往往忘记了照片背后的原因、情绪和故事。现有相册主要按时间和地点聚合，缺少对“意义”的主动捕捉；普通笔记应用又很难和照片、位置、3D 视觉资产自然结合。

随着移动端视觉理解、深度估计、3D 重建和多模态 Agent 能力成熟，用户可以从一张照片、一组照片或一段短视频中生成可旋转、可标注、可回看的 3D 记忆对象。这个产品可以成为旅行者的个人“意义博物馆”：每个展品不只是图片，而是可被再次进入的回忆。

## 4. 用户画像

### 4.1 核心用户

- 独自旅行者：希望把旅途中的感受、偶遇和细节认真保存下来。
- 深度旅行者：关注城市肌理、建筑、地方文化、物件和故事，不满足于普通打卡照。
- 内容创作者：需要把旅行素材整理成更有表现力的故事、游记或展览。
- 留学生、长期旅居者：希望记录跨城市、跨国家生活中的个人记忆。

### 4.2 用户需求

- 快速记录：在旅途中以最少操作保存照片、地点和想法。
- 意义补全：系统主动帮用户补上“为什么这件事重要”。
- 3D 转化：将重要照片或物件生成可交互的 3D 记忆对象。
- 易于回看：通过地图、时间线、标签、地点和语义搜索找回记忆。
- 可控分享：可以导出游记、3D 卡片、网页展览或私密链接。
- 私密可信：旅行记录、位置、人物和个人故事需要清晰的隐私控制。

## 5. 产品目标

### 5.1 核心目标

1. 帮助用户在旅行中低成本记录有意义的瞬间。
2. 将照片、说明、地点、时间、情绪和 3D 对象组织成结构化记忆。
3. 用 Agent 主动提问、整理和总结，减少用户事后整理负担。
4. 支持用户用地图、时间线和 3D 记忆库重新进入旅行经历。

### 5.2 成功指标

- 首次旅行记录完成率：用户创建首个旅行项目并添加至少 3 条记忆。
- 3D 生成完成率：提交 3D 生成任务后成功得到可查看结果的比例。
- 说明补全率：有文本、语音或 Agent 问答说明的记忆占比。
- 回访率：旅行结束 7 天后仍打开查看或编辑记录的用户比例。
- 分享率：导出游记、3D 卡片或分享链接的旅行项目占比。
- 用户主观满意度：用户认为“这比普通相册更能保存回忆”的评分。

## 6. 非目标

- 不做完整 OTA、酒店、机票、行程预订。
- 不以大众点评式地点评价为主要方向。
- 不把产品定位为公开社交平台，MVP 以私人记录和可控分享为主。
- 不要求所有照片都生成真实完整 3D；根据素材质量提供不同层级的 3D 或 2.5D 表现。
- 不承诺对人物、私有建筑、敏感地点进行自动公开展示，默认遵循隐私优先。

## 7. 核心用户旅程

### 7.1 旅途中快速记录

1. 用户创建一次旅行，例如“京都 2026 春”。
2. 用户拍摄一张有意义的照片，或从相册导入。
3. Agent 自动读取时间、地点、照片内容和可能的场景。
4. 用户选择“保存为记忆”。
5. Agent 用 1 到 3 个简短问题引导用户补充意义，例如：
   - “你为什么想留下这个地方？”
   - “这张照片里最重要的是哪个物件或细节？”
   - “当时发生了什么？”
6. 用户通过文字或语音添加说明。
7. 记忆被保存到本次旅行的时间线和地图中。

### 7.2 将照片生成 3D 记忆对象

1. 用户在某条记忆中点击“生成 3D”。
2. 系统判断素材类型：
   - 单张照片：生成深度感较强的 2.5D 视差视图或简化 3D 场景。
   - 多角度照片：生成更完整的 3D 模型、点云、mesh 或 Gaussian Splatting 表示。
   - 短视频：抽帧并重建可浏览场景。
3. 用户选择主要对象或区域，例如建筑立面、雕塑、纪念品、街角小店。
4. 后台生成 3D 资产，并显示进度、失败原因和重试建议。
5. 生成完成后，用户可以旋转、缩放、添加标注点和说明。
6. 3D 对象成为该记忆的一部分，可在旅行展览视图中查看。

### 7.3 旅行结束后的整理

1. Agent 汇总本次旅行的地点、时间、照片和记忆说明。
2. 系统生成旅行回顾草稿，包括每日摘要、重要地点、代表照片、3D 记忆对象。
3. 用户编辑标题、故事顺序和展示方式。
4. 用户导出为私密网页、Markdown 游记、PDF 或可分享链接。

## 8. 信息架构

### 8.1 顶层对象

- 用户：账户、偏好、隐私设置、设备同步状态。
- 旅行：一次旅行项目，包含时间范围、地点范围、封面、成员和权限。
- 记忆：旅行中的单个有意义事件、地点、照片或物件。
- 媒体：照片、视频、音频、缩略图、生成图、3D 资产。
- 3D 对象：从照片、照片组或视频生成的可交互资产。
- 标注：附着在照片或 3D 对象上的说明点。
- Agent 对话：围绕旅行或记忆的问答、补全、总结和编辑历史。

### 8.2 主要视图

- 今日记录：旅途中快速添加和补全记忆。
- 旅行时间线：按时间展示每天的照片、说明、地点和 3D 对象。
- 地图视图：按地理位置查看记忆。
- 3D 记忆库：集中展示已生成的 3D 对象。
- 记忆详情页：照片、3D、说明、地点、标签、情绪和 Agent 对话。
- 旅行回顾页：自动生成的游记和可分享展览。
- 设置与隐私：同步、导出、权限、位置和人物隐私控制。

## 10. Agent 能力设计

### 10.1 Agent 角色

Agent 应表现为旅行记录助手，而不是替用户编造故事的作者。它的职责包括：

- 观察：理解照片、时间、地点和已有记录。
- 追问：用少量问题帮助用户补全意义。
- 整理：把碎片描述变成可回看的结构。
- 建议：提醒用户哪些照片适合 3D，哪些需要补拍。
- 总结：旅行结束后生成回顾草稿。
- 保护：提醒用户敏感位置、人物隐私和分享风险。

### 10.2 Agent 行为原则

- 不虚构事实：无法确认的信息必须标记为推测或询问用户。
- 少打扰：旅途中默认短问短答，不进行长篇访谈。
- 保留原始表达：用户的原话和 Agent 改写版本都应可追溯。
- 隐私优先：涉及人物、住址、证件、儿童、私密地点时默认不公开。
- 可撤销：Agent 自动生成的标题、标签、摘要和描述都可编辑或删除。

### 10.3 典型 Agent 提示

- “这张照片里你最想记住的是哪个细节？”
- “我看到这里像是一家小店门口，要不要记录它为什么特别？”
- “这组照片角度比较完整，可以尝试生成 3D 记忆对象。”
- “这张照片包含清晰人脸，分享时是否自动模糊？”
- “我把今天的记录整理成了 5 个片段，你想按时间还是按主题查看？”

## 11. 3D 生成产品方案

### 11.1 生成层级

- L1：单图深度视差。适用于单张照片，生成轻量 2.5D 体验，速度快，质量稳定。
- L2：主体 3D 卡片。适用于物件、雕塑、建筑细节等主体明确的照片，生成可旋转的近似 3D 展示。
- L3：多图/视频 3D 重建。适用于用户围绕对象拍摄多角度素材，生成更完整的点云、mesh 或 Gaussian Splatting 资产。
- L4：场景级 3D 回忆。适用于街角、房间、展馆等空间场景，生成可漫游的轻量场景。

### 11.2 素材质量判断

系统应在生成前判断：

- 是否有足够清晰度。
- 主体是否被遮挡。
- 光照是否过暗或过曝。
- 多图之间是否有足够视角差。
- 视频是否有严重运动模糊。
- 是否包含敏感人物或隐私信息。

### 11.3 生成流程

1. 上传或选择素材。
2. 进行素材质量检测。
3. 用户确认主体或区域。
4. 创建生成任务。
5. 后台执行深度估计、分割、重建、纹理生成、压缩和预览图生成。
6. 返回可查看资产和质量报告。
7. 用户添加 3D 标注和说明。
8. 资产绑定到记忆对象。

### 11.4 失败处理

常见失败原因应被翻译成用户可理解的提示：

- “角度太少，无法生成完整 3D。你可以补拍左侧、右侧和背面。”
- “主体边缘不清楚，建议选择更清晰的照片。”
- “照片太暗，当前只能生成基础深度效果。”
- “视频移动过快，抽帧结果不稳定。”

## 13. 系统结构草案

### 13.1 客户端

- 移动端：拍照、导入、录音、离线草稿、地图、时间线、3D 查看。
- 桌面端/Web：深度整理、编辑游记、批量管理、导出和分享。
- 3D Viewer：统一承载 2.5D、mesh、point cloud、Gaussian Splatting 等资产。

### 13.2 后端服务

- 用户与权限服务：账户、登录、隐私和分享权限。
- 旅行与记忆服务：Trip、Memory、Media、Annotation 的 CRUD。
- 媒体服务：上传、缩略图、EXIF、转码、存储。
- Agent 服务：多模态理解、提问、摘要、标签、语义检索。
- 3D 生成服务：任务队列、素材检测、重建管线、资产存储。
- 导出服务：卡片、网页、Markdown、PDF。

### 13.3 AI/3D 管线

- 图像理解：主体识别、场景识别、OCR、地标线索。
- 语音处理：语音转文字、说话内容摘要。
- 文本处理：标题、标签、摘要、游记生成。
- 向量索引：照片描述、用户说明、地点和标签的语义搜索。
- 深度估计：单图视差或 2.5D 体验。
- 主体分割：选择要生成 3D 的对象或区域。
- 3D 重建：多图、视频或场景重建。
- 资产优化：压缩、纹理、LOD、移动端预览。

## 14. 隐私与安全

- 默认所有旅行记录为 private。
- 分享链接默认不可搜索、可随时撤销。
- 用户可选择隐藏精确 GPS，仅展示城市级位置。
- 人脸、车牌、证件、住址等敏感内容在分享前提供自动检测和模糊。
- 原始照片、用户说明和 Agent 生成内容需要明确区分。
- 用户可以导出全部数据，也可以永久删除旅行或单条记忆。
- 3D 生成任务应说明是否会上传云端处理；如支持本地处理，需要在设置中明确标注。

## 18. 验收标准

### 18.1 记录闭环

- 用户能在 1 分钟内创建旅行并保存第一条带照片的记忆。
- 用户能为记忆添加说明、地点、标签和情绪。
- 用户能在时间线和地图中找回该记忆。

### 18.2 3D 闭环

- 用户能从记忆详情页发起 3D/2.5D 生成。
- 生成过程有状态提示。
- 成功后能打开查看器并进行基本交互。
- 用户能给 3D 对象添加至少一个说明点。

### 18.3 Agent 闭环

- Agent 能基于照片和说明生成标题、摘要和标签。
- Agent 提出的问题能帮助用户补充意义。
- 用户可以编辑或删除 Agent 生成内容。

### 18.4 分享闭环

- 用户能导出包含照片、说明、地点和时间的图片卡片。
- 私密分享状态清晰可见。
- 用户能撤销分享。

## 19. 风险与挑战

- 3D 生成质量不稳定：需要用 L1 到 L4 分层体验降低用户预期落差。
- 移动端性能压力：3D 资产需要压缩、LOD 和渐进式加载。
- 用户旅途中不愿打字：必须提供语音、短问题和稍后补全。
- Agent 过度生成导致不真实：必须保留用户原话并标注生成内容。
- 隐私风险高：位置、人脸和个人故事需要默认私密。
- 成本控制：3D 生成和多模态理解可能成本较高，需要任务配额、压缩和异步队列。

## 20. 待确认问题

- 第一版主要面向移动端 App、Web App，还是桌面端优先？
- 3D 生成希望优先追求真实重建，还是优先追求快速、稳定、有氛围的 2.5D 记忆效果？
- 用户是否需要完全离线可用，还是允许云端生成 3D？
- 记录对象是否主要是地点、物件、建筑，还是也包含人物故事？
- 分享是否只是私密链接，还是未来会发展成公开作品集？
- 数据是否需要兼容现有相册、Google Photos、Apple Photos 或 Notion/Obsidian？
